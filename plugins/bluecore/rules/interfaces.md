# インターフェース設計標準

## モジュール/コンポーネント設計

- 単一責任を保つ
- 公開インターフェースの入出力を明確にする
- 再利用するロジックは抽出する
- 状態は必要最小限の範囲に閉じ込める

## 条件ロジック

- 複雑な条件は名前付き変数に抽出する
- ネストした三項演算子を書かない

## API と入力検証

- REST のメソッドとリソースを対応させる
- レスポンスの形を一貫させる
- 入力はスキーマで検証する
- 検証エラーは内部情報を含めない安全なメッセージで返す
