# 信頼性標準

## エラーハンドリング

- 失敗しうる処理は明示的に扱う
- 例外・エラーを握りつぶさない
- ユーザー向けメッセージと内部ログを分ける

## 非同期処理

- 独立した処理は並列実行する
- 依存のある処理だけ順次実行する
- 並列結果はまとめて待ってから次段へ進む

## 型安全性

- 引数と戻り値に型を明示する
- 公開 API に曖昧な型を使わない
- データ構造は型定義/インターフェースで表現する
